Desenvolvedor

Desenvolvimento acessivel

Codigo semantico, estruturado e compativel com tecnologias assistivas.

Descricao de imagens
Atributo alt correto para cada tipo de imagem.
  • Imagens com conteudo (fotos, graficos, icones funcionais) devem ter alt descritivo.
  • Imagens decorativas usam alt="" para serem ignoradas por leitores de tela.
  • Prefira inserir imagens decorativas via CSS.
  • O alt nao aparece visualmente — e lido quando o usuario navega pelo teclado.
<!-- Imagem com conteudo --> <img src="grafico.png" alt="Grafico de barras: vendas cresceram 30% em 2024"> <!-- Imagem decorativa --> <img src="divisor.svg" alt="" role="presentation">
Hierarquia de cabecalhos
Estrutura logica com h1–h6 para navegacao rapida.
  • Use apenas um <h1> por pagina — titulo principal.
  • Nao pule niveis: de h2 direto para h4 e incorreto.
  • Nao use cabecalhos apenas para estilo — use CSS para formatacao visual.
  • Leitores de tela listam cabecalhos para navegacao: a hierarquia deve ser logica.
<h1>Titulo da pagina</h1> <h2>Secao principal</h2> <h3>Subsecao</h3> <h2>Outra secao</h2>
Links e atalhos de navegacao
Links descritivos e atalhos para pular blocos repetitivos.
  • O texto do link deve descrever o destino mesmo fora do contexto do paragrafo.
  • Inclua link “Ir para o conteudo” visivel ao foco (skip navigation).
  • Links externos devem indicar isso visualmente e via aria-label.
  • O estado :focus deve ser visivel com contraste minimo 3:1.
<!-- Skip link --> <a href="#conteudo" class="skip-link">Ir para o conteudo</a> <!-- Link externo --> <a href="..." aria-label="WCAG 2.2 (abre nova aba)" target="_blank">WCAG 2.2 ↗</a>
Estrutura de formularios
Elementos e associacoes corretas para campos acessiveis.
  • Cada campo deve ter um <label> associado via for/id.
  • Use <fieldset> + <legend> para agrupar radios e checkboxes.
  • Mensagens de erro devem ser associadas ao campo via aria-describedby.
  • Use o atributo autocomplete em campos comuns (nome, e-mail, endereco).
<label for="email">E-mail</label> <input type="email" id="email" aria-describedby="email-erro" autocomplete="email"> <span id="email-erro" role="alert">E-mail invalido</span>
HTML semantico e padroes W3C
Codigo valido, limpo e com elementos corretos.
  • Use elementos semanticos: <nav>, <main>, <article>, <aside>, <footer>.
  • Evite elementos descontinuados: <font>, <center>, <marquee>.
  • Declare o idioma: <html lang="pt-BR">.
  • Valide o HTML em validator.w3.org.
  • Configure <meta charset="UTF-8"> e <meta name="viewport">.
WAI-ARIA para conteudo dinamico
Roles, states e properties para componentes interativos.
  • Use aria-live para anunciar atualizacoes dinamicas (alertas, buscas).
  • Modais, abas e acordeoes precisam de role, aria-expanded e gestao de foco.
  • Prefira HTML nativo — use ARIA apenas quando necessario.
  • Nunca remova o outline de foco sem substituir por algo visivel.
<button aria-expanded="false" aria-controls="menu-principal"> Menu </button> <div aria-live="polite" aria-atomic="true"> <!-- conteudo dinamico --> </div>
Designer

Design inclusivo

Cor, tipografia, layout e interacao que funcionam para todos.

Cores e contraste
Proporcoes minimas conforme WCAG 2.2 nivel AA.
  • Texto normal: contraste minimo 4,5:1 em relacao ao fundo.
  • Texto grande (18px+ ou 14px+ negrito): minimo 3:1.
  • Elementos de UI e graficos informativos: minimo 3:1.
  • Nunca use cor como unico meio de transmitir informacao — combine com icone ou texto.
  • Ferramenta: WebAIM Contrast Checker.
Tipografia legivel
Tamanhos e formatacoes que facilitam a leitura.
  • Corpo de texto: minimo 16px — use rem para respeitar o zoom do navegador.
  • line-height minimo de 1,5 para o corpo de texto.
  • Evite italico em blocos de texto — dificulta leitura para dislexia.
  • Nao justifique texto — o espacamento irregular cria dificuldade de leitura.
  • Prefira alinhamento a esquerda.
Links identificaveis
Diferenciacao clara entre links e texto comum.
  • Links inline devem ter sublinhado ou outro indicador alem da cor.
  • Links externos: indicacao visual (↗) e anuncio para leitores de tela.
  • Estado :focus visivel com outline de pelo menos 2px e contraste 3:1.
  • Area de toque minima: 44×44px (WCAG 2.5.5).
Icones e elementos informativos
Todo icone funcional precisa de rotulo textual.
  • Icones que transmitem informacao devem ter texto visivel ou aria-label.
  • Icones decorativos devem ter aria-hidden="true".
  • Nao confie so em cor para estados: sucesso, erro, aviso — use icone + texto.
  • Tooltips devem ser acessiveis via teclado e leitores de tela.
Espacamento e layout
Relacoes visuais claras entre elementos.
  • Espacamento consistente deixa claro quais conteudos sao relacionados.
  • letter-spacing minimo 0,12em; word-spacing minimo 0,16em.
  • Layout responsivo: funcionar com zoom de 400% sem scroll horizontal.
  • Largura ideal de coluna de texto: 70–80 caracteres (~600–700px).
Animacoes e movimento
Controle de movimento para quem tem sensibilidade vestibular.
  • O usuario deve poder pausar, parar ou ocultar animacoes automaticas.
  • Use prefers-reduced-motion para desativar animacoes configuradas no SO.
  • Conteudo que pisca mais de 3 vezes por segundo pode causar convulsoes — evite.
@media (prefers-reduced-motion: reduce) { * { animation-duration: 0.01ms !important; transition-duration: 0.01ms !important; } }
Conteudo

Texto, midia e linguagem

Conteudo claro, inclusivo e compativel com diferentes necessidades.

Descricao de imagens e midia
Textos alternativos para todo conteudo nao-textual.
  • Fotos, ilustracoes, tabelas, graficos e GIFs devem ter descricao textual.
  • Evite redundancia: “A foto ilustra” e pleonasmo — seja direto e objetivo.
  • Evite adjetivos de juizo de valor (bonito, feio, bom, mau).
Formula para descrever imagens
Formato + Sujeito + Cenario + Contexto + Acao
Completa: o que/quem + onde + como + faz o que + como + quando + de onde
Video e audio acessiveis
Legendas, Libras e transcricoes para todos.
  • Todo video com fala deve ter legenda (para surdos alfabetizados em portugues).
  • Inclua janela de Libras — avatar ou interprete — para surdos nao oralizados.
  • Podcasts e audios devem ter transcricao em texto.
  • Videos devem ter audiodescricao que contextualize sem atrapalhar o audio original.
Linguagem clara e simples
Textos compreensiveis para o maior numero de pessoas.
  • Use frases curtas, paragrafos objetivos e ordem direta (sujeito + verbo + complemento).
  • Prefira voz ativa a passiva.
  • Evite figuras de linguagem, girias e termos tecnicos sem explicacao.
  • Na primeira ocorrencia, escreva siglas por extenso.
Links descritivos no texto
Textos de link que facam sentido fora do contexto.
  • Evite “clique aqui”, “saiba mais” isolados — cegos navegam listando so os links.
  • Prefira: “Acesse o site [nome]” ou “Leia o relatorio de acessibilidade 2024”.

Redes sociais
Boas praticas para posts acessiveis.
  • Adicione texto alternativo as imagens postadas — Instagram, Facebook e X oferecem esse recurso.
  • Use CamelCase em hashtags: #AcessibilidadeDigital em vez de #acessibilidadedigital.
  • Descreva GIFs e stickers informativos na legenda da publicacao.
  • Evite excesso de emojis no meio do texto — leitores de tela os verbalizam todos.

Checklist rapido para o dia a dia

Desenvolvedor
  • Todas as imagens tem alt correto?
  • Existe so um <h1> e a hierarquia e logica?
  • Todos os campos de formulario tem <label>?
  • O site funciona 100% pelo teclado?
  • A pagina tem lang declarado?
  • O HTML passou no validador W3C?
  • Componentes dinamicos usam ARIA correto?
  • Skip link esta implementado?
Designer
  • Contraste de texto ≥ 4,5:1 (ou 3:1 para texto grande)?
  • Links se diferenciam do texto alem da cor?
  • Area de toque minima 44×44px?
  • Estado de foco visivel e com contraste adequado?
  • Animacoes podem ser desativadas?
  • Icones funcionais tem rotulo textual?
  • Layout funciona com zoom de 200%?
Conteudo
  • Todas as imagens tem descricao contextual?
  • Videos tem legenda e janela de Libras?
  • Podcasts tem transcricao em texto?
  • Links tem texto descritivo do destino?
  • Siglas foram explicadas na primeira ocorrencia?
  • Hashtags usam CamelCase?
